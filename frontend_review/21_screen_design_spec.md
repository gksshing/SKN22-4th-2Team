## 📄 Screen 0 — 공통: 글로벌 헤더 (GNB)

### 개요
모든 화면 최상단에 고정 표시되는 공통 헤더.

### 레이아웃

| 좌측 | 우측 |
|---|---|
| `💡 쇼특허 (Short-Cut) AI` 로고 + 서비스 설명 문구 | 비로그인: `로그인` / `회원가입` 버튼<br>로그인 후: `{이메일}님` + `로그아웃` 버튼 |

### 상태 분기

| 상태 | 표시 내용 |
|---|---|
| 비로그인 | 로그인(텍스트) / 회원가입(파란 Pills 버튼) |
| 로그인됨 | 이메일 텍스트 + 로그아웃(테두리 버튼) |

### 인터랙션
- `로그인` 클릭 → AuthGuard 오버레이(로그인 폼) 노출
- `회원가입` 클릭 → AuthGuard 오버레이(회원가입 폼) 노출
- `로그아웃` 클릭 → 세션 초기화, 비로그인 상태로 전환

## 📄 Screen 1 — 로그인 / 회원가입 (AuthGuard)

### 개요
비로그인 사용자가 분석 기능 접근 시 또는 헤더 버튼 클릭 시 표시되는 오버레이 인증 화면.

### 레이아웃 (탭 전환 방식)

```
┌────────────────────────────────────┐
│         [로그인] | [회원가입]        │  ← 탭 전환
├────────────────────────────────────┤
│  이메일 ____________________________│
│  비밀번호 __________________________│  (👁 토글 아이콘)
│  비밀번호 강도 바 (회원가입만 표시)   │
│  [로그인 / 회원가입] 버튼            │
│  에러 메시지 (AuthErrorToast)       │
└────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 역할 |
|---|---|
| `AuthGuard.tsx` | 인증 여부 판단, 오버레이 제어 |
| `LoginForm.tsx` | 이메일/비밀번호 입력, 로그인 요청 |
| `SignupForm.tsx` | 이메일/비밀번호/확인 입력, 회원가입 요청 |
| `PasswordToggleInput.tsx` | 비밀번호 표시/숨김 토글 |
| `PasswordStrengthBar.tsx` | 비밀번호 강도 시각화 (회원가입 전용) |
| `AuthErrorToast.tsx` | 로그인/가입 실패 에러 토스트 |
| `SessionExpiredToast.tsx` | 세션 만료 시 안내 토스트 |

### 유효성 검사 규칙

| 필드 | 조건 |
|---|---|
| 이메일 | RFC 이메일 형식 |
| 비밀번호 | 최소 8자, 영문+숫자 조합 |
| 비밀번호 확인 | 비밀번호와 일치 여부 (회원가입) |

### 상태 분기

| 상태 | 화면 |
|---|---|
| 입력 중 | 폼 정상 표시 |
| 제출 중 | 버튼 비활성화 + 로딩 스피너 |
| 에러 | AuthErrorToast 노출 |
| 성공 | 오버레이 닫힘, 헤더에 사용자 이메일 표시 |

## 📄 Screen 2 — 메인 분석 화면 (분석 전)

### 개요
로그인 후 처음 보이는 핵심 화면. 아이디어 입력 + IPC 분류 필터 + 히스토리 사이드바 구성.

### 레이아웃 (2열 구조)

```
┌──────────────────┬───────────────────────────────────┐
│  히스토리         │  IPC 카테고리 필터                 │
│  사이드바         │  ┌─────────────────────────────┐  │
│                  │  │  IpcFilterSelector          │  │
│  [과거 분석 목록] │  └─────────────────────────────┘  │
│                  │  아이디어 입력 텍스트에어리어        │
│  HistorySidebar  │  [🔍 특허 침해 분석 시작] 버튼      │
│                  │  글자 수 카운터 (X / 2000)         │
└──────────────────┴───────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 역할 |
|---|---|
| `HistorySidebar.tsx` | 과거 분석 내역 목록 표시, 원클릭 재분석 |
| `HistoryItems.tsx` | 개별 히스토리 카드 + 빈 상태 Placeholder |
| `IpcFilterSelector.tsx` | 특허 IPC 분류 코드 필터 선택 (다중 선택) |
| `IdeaInput.tsx` | 아이디어 텍스트 입력폼 (20~2000자) |

### IPC 필터 상세

| 항목 | 내용 |
|---|---|
| 선택 방식 | 체크박스 다중 선택 |
| 적용 효과 | 선택한 IPC 분류 내에서만 유사 특허 검색 |
| 미선택 시 | 전체 특허 DB 대상 검색 |
| UI | 선택한 필터가 태그(Chip) 형태로 표시 |

### 히스토리 사이드바 상세

| 항목 | 내용 |
|---|---|
| 데이터 소스 | `GET /api/history` (X-Session-ID 헤더 기반) |
| 카드 내용 | 위험도 배지 + 아이디어 요약(50자) + 날짜 |
| 원클릭 동작 | 해당 아이디어로 IdeaInput 자동 채움 + 재분석 |
| 빈 상태 | "아직 분석 기록이 없습니다" + 입력 유도 메시지 |
| 갱신 시점 | 분석 완료(`isComplete` 전환) 시 자동 새로고침 |

## 📄 Screen 3 — 분석 진행 중 화면

### 개요
`🔍 분석 시작` 버튼 클릭 후 RAG + LLM 응답 대기 중 표시 (SSE 스트리밍).

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  ████████████████████░░░  75%               │  ← 프로그레스 바
│  단계 메시지: "특허 DB 유사도 계산 중..."    │  ← 단계별 텍스트
│                                             │
│  ┌──────────────────┐  (스켈레톤 카드 3개)  │
│  │ ░░░░░░░░░         │                     │
│  │ ░░░░░░░░░░░░░     │                     │
│  └──────────────────┘                      │
│                                             │
│  ───────────────────────────────────────── │
│  ⚠️ [30초 경과] — TimeoutToast (우하단)     │
└─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 역할 |
|---|---|
| `RagSkeleton.tsx` | 카드 스켈레톤 + 프로그레스 바 |
| `SkeletonLoader.tsx` | 헤더 스켈레톤 블록 |
| `ProgressStepper.tsx` | 단계별 메시지 및 스텝 표시 |
| `GlobalLoading.tsx` | 전역 로딩 오버레이 (선택적) |
| `TimeoutToast.tsx` | 30초 경과 시 우하단 경고 토스트 |

### 진행 단계 메시지

| 퍼센트 | 메시지 |
|---|---|
| 0% | "특허 DB 연결 중..." |
| 20% | "아이디어 분석 및 벡터 변환 중..." |
| 40% | "유사 특허 검색 중..." |
| 60% | "특허 DB 유사도 계산 중..." |
| 80% | "AI 침해 위험도 분석 중..." |
| 95% | "결과 정리 중..." |

### 타임아웃 토스트 (TimeoutToast)
- **발동 조건**: 분석 시작 후 30초 경과
- **표시 내용**: "응답이 지연되고 있습니다. 잠시 후 자동 취소됩니다."
- **위치**: 화면 우하단 고정

### 개요
RAG 분석 완료 후 표시되는 결과 화면.

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  📊 침해 위험도: [🔴 HIGH] 82%              │  ← 위험도 배지 + 점수
│  유사 특허 총 N건 발견                        │
├─────────────────────────────────────────────┤
│  유사도 분포 바 차트 (SimilarityBarChart)    │
├─────────────────────────────────────────────┤
│  ┌─────────────────────────────────┐        │
│  │  #1  KR10-2023-XXXXXXX [🔴 87%]│        │
│  │       특허 제목                  │        │
│  │       위험 사유 요약              │        │
│  │       [원문 보기 🔗]             │        │
│  └─────────────────────────────────┘        │
│  (유사 특허 카드 최대 5건)                   │
├─────────────────────────────────────────────┤
│  [PDF 내보내기]  [🔄 새 아이디어 분석]       │
└─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 역할 |
|---|---|
| `ResultView.tsx` | 결과 화면 전체 컨테이너 |
| `SimilarityBarChart.tsx` | 유사도 점수 가로 바 차트 시각화 |
| `exportPdf.ts` | `window.print()` 기반 PDF 내보내기 |

### 위험도 컬러 코딩

| 위험도 | 점수 | 색상 | 배지 |
|---|---|---|---|
| High | 80% 이상 | 🔴 Red | `bg-red-100 text-red-700` |
| Medium | 50~79% | 🟡 Yellow | `bg-yellow-100 text-yellow-700` |
| Low | 49% 이하 | 🟢 Green | `bg-green-100 text-green-700` |

### 유사 특허 카드 구성

| 항목 | 내용 |
|---|---|
| 순위 | #1 ~ #N |
| 특허 번호 | KR / US 번호 (모노스페이스 폰트) |
| 유사도 배지 | 컬러 코딩된 퍼센트 |
| 제목 | 최대 2줄 |
| 위험 사유 | 3줄 요약 (LLM 생성) |
| 원문 링크 | KIPRIS 등 외부 링크 (patent.url 제공 시) |

## 📄 Screen 5 — 에러 화면 (ErrorFallback)

### 개요
API 에러 또는 예외 상황 발생 시 표시되는 친화적 에러 화면.

### 에러 유형별 메시지

| 에러 코드 | 제목 | 메시지 | 조치 |
|---|---|---|---|
| `RATE_LIMITED` | 분석 한도에 도달했습니다 🚦 | 오늘의 분석 횟수를 모두 사용했습니다. | 내일 다시 이용 / 히스토리 보기 |
| `SESSION_EXPIRED` | 세션이 만료되었습니다 🔄 | 세션이 재발급되었습니다. | 다시 시도 버튼 |
| `TOKEN_EXCEEDED` | 입력 텍스트가 너무 깁니다 🚫 | 입력을 줄여 다시 시도하세요. | 재입력 버튼 |
| `TIMEOUT` | 분석 시간 초과 ⏱️ | 잠시 후 다시 시도해 주세요. | 다시 시도 버튼 |
| `NOT_FOUND` | 유사 특허 없음 📭 | 일치하는 특허가 없습니다. | 새 분석 버튼 |
| `NETWORK_ERROR` | 네트워크 연결 오류 🔌 | 서버 연결에 실패했습니다. | 다시 시도 버튼 |

### 레이아웃
```
┌──────────────────────────────────────┐
│           (에러 이모지)               │
│         에러 제목                     │
│         에러 설명 메시지              │
│  [다시 시도]         [새 아이디어]    │
└──────────────────────────────────────┘
```

## 📐 공통 UX 규칙

### 반응형 브레이크포인트

| 브레이크포인트 | 레이아웃 |
|---|---|
| 모바일 (< 768px) | 히스토리 사이드바 숨김, 단일 컬럼 |
| 태블릿 (768px~) | 히스토리 사이드바 표시, 2열 |
| 데스크탑 (1280px~) | 최대 너비 `max-w-7xl` 제한 |

### 상태 전환 플로우

```
[비로그인 상태]
    ↓ 로그인 버튼 클릭
[AuthGuard 오버레이]
    ↓ 로그인 성공
[메인 분석 화면] ──────────────────┐
    ↓ 아이디어 입력 + 분석 시작     │
[로딩 화면]                        │
    ↓ 분석 완료                    │
[결과 화면] → [새 아이디어 버튼] ──┘
    ↓ 에러 발생
[에러 화면] → [다시 시도] → [메인 분석 화면]
```

### 접근성 (a11y) 요구사항
- 모든 버튼에 `id` 속성 부여 (브라우저 테스트 지원)
- 에러 메시지는 `role="alert"` 적용
- 폼 입력 필드에 `aria-label` 또는 `<label>` 연결
- 키보드 Tab 순서: 헤더 → 필터 → 입력 → 제출

## 📌 백엔드 협업 필요 사항

| 항목 | 내용 |
|---|---|
| `GET /api/history` | `X-Session-ID` 헤더 기반 세션 조회 (쿼리 파라미터 X) |
| 429 응답 | `Retry-After` 헤더 포함 (카운트다운 타이머용) |
| `patent.url` | 유사 특허 원문 링크 필드 응답에 포함 필요 |
| 분석 응답 필드 | `riskLevel`, `riskScore`, `similarCount`, `uniqueness`, `topPatents[]` |
