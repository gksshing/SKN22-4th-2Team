### 🎨 UI/UX 개발 내용 요약 (모달 기반 인증 시스템 전환)

**1. 인증 방식 전환 (Full-screen → Modal)**

- **메인 페이지 유지**: 기존에는 비로그인 시 전체 화면이 로그인 폼으로 바뀌었으나, 이제는 홈 화면이 그대로 유지된 상태에서 부드러운 **모달(Modal) 오버레이**가 뜨도록 변경했습니다.
- **배경 블러 효과**: `backdrop-blur-sm`과 반투명 배경을 적용하여 콘텐츠에 집중할 수 있는 환경을 조성했습니다.
- **애니메이션 추가**: 모달 등장 시 `zoom-in` 및 `fade-in` 효과를 추가하여 자연스러운 전환을 구현했습니다.

**2. 헤더 및 네비게이션 개선**

- **로그인 버튼 디자인**: 헤더의 로그인 버튼을 더 둥글고(12px+) 강조된 블루 스타일로 수정하여 클릭률을 유도했습니다.
- **사용자 프로필**: 로그인 시 사용자 이메일이 태그 형태로 깔끔하게 표시되도록 인라인 디자인을 적용했습니다.

### 📋 운영 환경 장애(무응답) 분석 및 패치 제안

**장애 원인**:

- `apiClient.ts`에서 `VITE_API_BASE_URL`이 설정되지 않았을 경우 기본값인 `http://localhost:8000`으로 요청을 보냅니다.
- 운영 서버 브라우저에서 실행될 때, `localhost`는 사용자의 PC를 가리키므로 서버(ALB)와 통신되지 않아 '무응답' 상태가 됩니다.

**권장 사항 (DevOps 전달용)**:

1. `Dockerfile` 빌드 시 혹은 `.env` 파일에 `VITE_API_BASE_URL=` (빈 값) 혹은 `/` 를 설정하여 **상대 경로**로 통신하게 수정해야 합니다. (동일 오리진 서빙 중이므로 상대 경로가 가장 안전합니다)
2. ALB 보안 그룹에서 80/443 포트가 백엔드 8000 포트로 정상 포워딩되는지 확인이 필요합니다.
