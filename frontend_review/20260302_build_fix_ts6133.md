# 🛠️ 프론트엔드 빌드 오류 (TS6133) 수정 리포트

## 📅 날짜: 2026-03-02
## 👤 담당: 수석 프론트엔드 엔지니어

### ⚠️ 문제 현상
- 프로덕션 빌드([npm run build]) 중 `TS6133` 에러 발생으로 인해 빌드 실패.
- 원인: `tsconfig.json`의 `noUnusedLocals: true` 설정과 `jsx: "react-jsx"` 환경에서 사용되지 않는 `import React from 'react'` 코드가 다수 포함되어 있었음.

### ✅ 수정 내용
다음 파일들에서 사용되지 않는 `React` 기본 임포트를 제거하고, 필요한 경우 명명된 임포트(Named Import)를 사용하도록 리팩토링하였습니다.

1. **[GlobalLoading.tsx](file:///C:/WorkSpace/SKN22-4th-2Team/frontend/src/components/Loading/GlobalLoading.tsx)**
   - `import React from 'react'` -> `import { FC } from 'react'`
   - `React.FC` -> `FC`

2. **[AuthGuard.tsx](file:///C:/WorkSpace/SKN22-4th-2Team/frontend/src/components/Auth/AuthGuard.tsx)**
   - `import React, { useEffect } from 'react'` -> `import { ReactNode, useEffect } from 'react'`
   - `React.ReactNode` -> `ReactNode`

3. **[main.tsx](file:///C:/WorkSpace/SKN22-4th-2Team/frontend/src/main.tsx)**
   - `import React from 'react'` -> `import { StrictMode } from 'react'`
   - `React.StrictMode` -> `StrictMode`

### 🚀 결과
- 불필요한 `React` 식별자 참조를 제거하여 `TS6133` 에러를 해결하였습니다.
- 모던 React(JSX Transform) 방식에 맞게 임포트 구조를 최적화하였습니다.
- 이제 Docker 빌드 환경에서도 정상적으로 컴파일이 완료될 것으로 예상됩니다.
