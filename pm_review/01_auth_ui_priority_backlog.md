# 📊 쇼특허(Short-Cut) PM 브리핑: 인증 UI 노출 문제 진단 및 넥스트 백로그

## 🚨 현재 '로그인/회원가입 창이 안 보이는' 원인 진단
현재 개발팀(프론트엔드 에이전트)이 로그인 및 Auth Guard 로직을 완성한 곳은 **React 기반의 신규 프론트엔드(`frontend/` 폴더)**입니다. 하지만 현재 캡처해주신 화면은 기존의 프로토타입인 **Streamlit(`app.py`)** 화면입니다. 
따라서 신규 UI를 확인하시려면 Streamlit 서버(8501 포트)가 아닌, React 개발 서버(5173 포트)로 접속하셔야 합니다. 

---

## 📋 향후 문제 해결 및 고도화를 위한 우선순위 백로그 (Todo)

PM 관점에서 현재 구현된 기술적 태스크에 UX 및 예외 처리를 보완하여 다음 백로그를 구성했습니다. 모든 이슈는 현재 **'Todo'** 상태로 세팅할 것을 권장합니다.

### 🥇 Priority 1: 프론트엔드-백엔드 통합 및 실행 환경 전환 (Critical)
- **[Task] React 프론트엔드 구동 및 E2E 테스트**
  - Streamlit 환경을 종료하고 `npm run dev`를 통해 React 환경에서 로그인 창이 정상 노출되는지 확인.
  - FastAPI 백엔드와 통신하여 세션(Cookie)이 완벽히 저장되고 유지되는지 통합 테스트 진행.
- **[Task] CORS 설정 및 네트워크 오류 점검**
  - 프론트(5173)와 백엔드(8000) 간의 API 통신 시 CORS 에러가 없는지 최종 점검.

### 🥈 Priority 2: 예외/에러 UI 및 로딩 상태 고도화 (UX/기획)
- **[Issue] RAG 로딩 및 타임아웃 UX 처리**
  - LLM API 응답 대기 시간(Timeout 30초 등)이 초과되었을 때 화면이 멈추지 않도록 "친절한 에러 안내 모달"과 "재시도 버튼" 추가.
- **[Issue] 검색 결과 없음(Zero-result) Fallback UI**
  - 특허 검색 결과가 없을 때 빈 화면 대신 "다른 키워드로 검색해보세요" 등의 사용자 가이드(Empty State UI) 렌더링.

### 🥉 Priority 3: 사용자 인증 플로우 기능 확장 (UX/기획)
- **[Issue] 미인증 접근 시 리다이렉트 흐름 개선**
  - 사용자가 특정 분석 결과를 보려다 로그인 창으로 튕겼을 때, 로그인 직후 처음에 보려던 페이지로 다시 보내주는 UX(Redirect URL) 추가.
- **[Issue] 비밀번호 찾기(Reset Password) 플로우 기획**
  - 현재 로그인/회원가입은 있지만 비밀번호 분실 시 대응 시나리오 부재. 이메일 인증 기반 비밀번호 재설정 UI 설계.

**👨‍💻 Scrum Master의 제안:**
가장 첫 번째 단계로 터미널에서 `cd frontend && npm run dev` 명령어를 통해 React 화면을 띄워 완성된 인증 폼을 먼저 검수해 보시는 것을 추천드립니다. 해당 통합이 완료되었다고 알려주시면, 그 다음 가장 우선순위가 높은 **'에러 UI 고도화'** 이슈를 'In Progress'로 옮기고 작업을 지시하겠습니다!
