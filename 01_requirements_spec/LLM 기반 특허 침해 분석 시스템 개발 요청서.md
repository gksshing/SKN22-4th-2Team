# 📋 요청서 LLM 기반 특허 침해 분석 시스템 개발 요청서

### * 1. 프로젝트 배경 및 전략적 추진 방향

현대 기술 시장에서 지식재산권(IP)은 기업의 생존과 직결된 핵심 자산이나, 역설적으로 혁신의 속도를 저해하는 가장 큰 병목 구간이기도 합니다.
기존의 전통적인 특허 침해 분석 방식은 변리사 등 고비용 전문가의 수동 검토에 의존하며, 건당  평균 250만 원의 비용 과  수주 이상의 긴 소요 시간 을 요구합니다.
이는 자본과 시간이 희소한 스타트업에게 치명적인 진입 장벽으로 작용합니다.
본 프로젝트은 이러한 비효율을 AI 기술로 해결하여 비즈니스 민첩성을 극대화하는 것을 목표로 합니다.

**혁신성:**  기존 수동 프로세스를 LLM 및 RAG 기반 자동화 파이프라인으로 전환하여 검토 시간을 '분' 단위로 단축합니다.
**전략적 가치:** 단순히 비용을 절감하는 것을 넘어, 기업이 R&D 초기 단계에서 지식재산 기반의 전략적 피벗을 수행할 수 있는 환경을 제공합니다. 이는 아이디어를 주 단위로 검증하고 수정할 수 있는 초고속 R&D 사이클 구축을 가능케 합니다.서비스의 전략적 가치를 바탕으로, 이를 실현하기 위한 구체적인 서비스 정의와 목표를 다음 장에서 상세히 기술합니다.

### * 2. 서비스 개요 및 비즈니스 목표

특허앱은 사용자의 아이디어를 입력받아 AI가 기존 특허와의 중복/침해 리스크를 실시간 분석하고, 근거 기반의 시각화 리포트를 제공하는 지능형 IP 솔루션입니다.

**2.1 주요 타겟 사용자별 기대 효과**

타겟 사용자,기대 효과
창업 준비자 / 발명가,아이디어 구상 단계에서 즉각적인 특허성 검토를 통한 사업 방향성 조기 확립
스타트업,R&D 초기 IP 리스크 관리 비용의 획기적 절감 및 투자 유치 시 기술 보호 입증
기업 R&D 담당자,방대한 특허 문헌 사이에서 유관 특허를 신속하게 필터링하여 기획 업무 효율성 극대화

**2.2 정량적/정성적 목표**

정량적 목표:  분석 응답 시간 30초 이내 달성, 일일 최대 10회 분석 기회 제공, 동시 접속자 50명 수용 가능한 고가용성 인프라 구축.
정성적 목표:  비전문가도 즉시 이해 가능한 직관적 리스크 지표 제공 및 검색된 컨텍스트 내에서의 답변 제한을 통한 데이터 신뢰도 확보.

### 3. AI 특허 분석 파이프라인 및 RAG 엔진 상세 요구사항

본 시스템의 핵심 경쟁력은 대규모 특허 데이터에서 정확한 문맥을 추출하고 논리적 분석을 수행하는 RAG(검색 증강 생성) 파이프라인에 있습니다.

**3.1 하이브리드 검색 및 IPC 필터링 최적화**

Pinecone 벡터 DB를 활용하여  Hybrid Search(Dense + BM25)  방식을 채택하며, 검색 효율성을 극대화하기 위한 전략을 적용합니다.
IPC Metadata Pre-filtering:  사용자가 선택한 국제특허분류(IPC) 코드를 기반으로 검색 범위를 사전에 한정합니다. 이는 벡터 검색 공간을 압축하여 검색 속도를 향상시키고, 무관한 기술 분야의 혼입을 막아 분석의 정확도를 높이며 컴퓨팅 비용을 최적화하는 핵심 전략입니다.
Hybrid Search:  아이디어의 의미론적 유사성(Dense)과 기술적 키워드 매칭(BM25)을 결합하여 누락 없는 유사 특허 추출을 보장합니다.

**3.2 환각 현상 방지 및 보안 가드레일**

Grounded Response Principle:  LLM이 반드시 '검색된 컨텍스트 내의 특허 문서'만을 근거로 답변하도록 제한하는 프롬프트 엔지니어링을 적용합니다. 근거 없는 법적 해석이나 가공의 특허 번호 생성을 원천 차단합니다.
Prompt Injection Defense:  분석 엔진 인입 전 프롬프트 인젝션 의심 입력을 감지하고 거부하는 보안 레이어를 배치하여 모델의 무결성을 보호합니다.

**3.3 분석 결과 리포트 구성**

위험도 배지:  High(Red), Medium(Yellow), Low(Blue) 컬러 코딩을 통한 직관적 리스크 표기.
유사도 점수:  분석된 특허와의 정량적 유사 수치를 바 차트로 시각화.
유사 특허 카드:  특허 번호, 발명의 명칭, 위험 사유 요약, 그리고 검증을 위한 원문 링크를 반드시 포함.

### * 4. 기능 요구사항 및 유즈케이스

시스템은 사용자 경험(UX)과 안정성을 고려한 4대 핵심 모듈로 구성됩니다.

**4.1 핵심 모듈별 상세 명세**

회원 인증:  이메일 기반 가입/로그인을 지원하며, 비밀번호는  bcrypt 로 해싱 저장합니다. 보안 강도 시각화 바를 제공하여 사용자 보안 인식을 제고하며, 세션 만료 시 명확한 재로그인 가이드를 제공합니다.
**아이디어 분석:**  20자~2000자 사이의 텍스트 입력을 처리합니다.  SSE 스트리밍  방식을 적용하여 단계별 진행 상황(예: 유사 특허 검색 중..., 분석 리포트 생성 중...)을 실시간 표시합니다. 이는 LLM의 추론 시간 동안 발생하는 사용자 이탈을 방지하고, ALB의 타임아웃을 방지하는 기술적 이점을 제공합니다.
**결과 시각화:**  분석 완료 후 위험도 및 상세 카드를 렌더링하며, 브라우저 인쇄 기능을 활용한 분석 결과 PDF 내보내기를 지원합니다.
**히스토리 관리:**  사이드바를 통해 과거 분석 내역을 최신순으로 제공합니다. 각 카드는 위험도 배지와 50자 요약을 포함하며,  X-Session-ID  헤더를 기반으로 세션별 내역을 안전하게 관리합니다.

**4.2 운영 정책 및 안정성 설계**

**사용량 제한:**  사용자/세션당 일일 분석 횟수를 10회로 제한합니다. 제한 초과 시 429 Too Many Requests 응답과 함께  Retry-After  헤더를 포함하여 재시도 가능 시간을 안내합니다.
**상태 유지:**  Rate Limit 상태는  Redis 에 영속화하여 서버 재시작 시에도 정책이 일관되게 유지되도록 합니다.

### * 5. 기술 스택 및 비기능 요구사항

**5.1 기술 스택 구성 및 선정 이유**

레이어,기술 스택,Interface / Protocol,선정 사유
Frontend,React,REST API / SSE,컴포넌트 기반 UI 개발 및 동적 분석 상태 렌더링 최적화
Backend,FastAPI,Pydantic / SQLAlchemy,비동기 처리 성능 및 AI 모델 API 연동 효율성 극대화
Infrastructure,AWS ECS Fargate,Docker (Non-root),서버리스 컨테이너 운영으로 관리 부담 감소 및 보안 강화
Database,"PostgreSQL, Redis",SQL / Key-Value,정형 데이터 관리 및 고속 캐싱의 상호 운용성

**5.2 보안 및 품질 보증**

**보안 레이어:**  API 키 등 민감 정보는  AWS Secrets Manager 에서 관리하여 소스코드 유출 리스크를 원천 차단합니다. 컨테이너는  non-root 사용자 로 실행하여 권한 탈취 시 피해를 최소화하며, 프론트엔드 출력 전  DOMPurify 를 통한 Sanitize로 XSS 공격을 방어합니다.
**성능 최적화:**  50명 동시 접속 시에도 30초 이내 응답을 보장하기 위해 Pinecone 검색을 3초 이내로 최적화하고, Docker 멀티 스테이지 빌드로 배포 효율성을 높입니다.
**유지보수성:**  모든 API 입출력은  Pydantic 모델 로 검증하며, Python  Type Hints 와 PEP8 표준을 준수하여 코드 품질을 관리합니다.

### * 6. 운영 제약 사항 및 향후 고도화 로드맵

**6.1 비용 최적화: 이중 모델 전략**

OpenAI API 비용의 효율적 관리를 위해 지능형 라우팅 전략을 제안합니다.
Fast 모델 (GPT-4o-mini):  초기 아이디어 요약, 임베딩 필터링, 단순 키워드 추출에 활용.
Full 모델 (GPT-4o):  심층적인 침해 리스크 분석 및 최종 리포트 생성에만 선별적으로 활용.
기대 효과:  전체 토큰 비용을 최대  80% 절감 하면서도 분석의 깊이를 유지합니다.

**6.2 잠재적 리스크 및 대응책**

데이터 동기화:  KIPRIS 기반 데이터는 실시간 DB와 지연이 발생할 수 있음을 사용자에게 명시(Disclaimer)합니다.개인정보 보호:  입력된 아이디어의 중요성을 고려하여, 향후 아이디어 입력값의 암호화 저장 옵션을 검토합니다.

**6.3 향후 고도화 우선순위**

**보안 및 신뢰성 (P0):**  XSS 방지(DOMPurify) 적용 및 429 에러 시 Retry-After 헤더 구현.
**데이터 무결성 (P0):**  X-Session-ID 헤더 기반의 히스토리 조회 API 완성.
**관리 효율화 (P1):**  MVP 이후 사용량 통계 및 사용자 관리를 위한 전용 관리자 대시보드 구축.본 요청서에 명시된 요구사항은 Short-Cut AI의 시장 안착을 위한 최소 요건이며, 기술 파트너와의 긴밀한 협력을 통해 완성될 예정입니다.

## 10. 유스케이스 시나리오

### UC-01. 신규 사용자 특허 분석

```
1. 사용자가 서비스에 접속한다.
2. 회원가입 버튼을 클릭하여 이메일/비밀번호로 가입한다.
3. 로그인 후 아이디어 입력창에 아이디어를 작성한다.
4. (선택) IPC 분류 필터를 선택한다.
5. '분석 시작' 버튼을 클릭한다.
6. 진행 중 단계별 메시지를 확인한다. (로딩 UI)
7. 분석 완료 후 위험도 배지 + 유사 특허 목록을 확인한다.
8. 필요 시 PDF로 내보낸다.
```

### UC-02. 재분석 (히스토리 선택)

```
1. 로그인한 사용자가 좌측 히스토리 사이드바에서 과거 분석 항목을 클릭한다.
2. 해당 아이디어가 입력창에 자동으로 채워지고 즉시 재분석이 실행된다.
3. 최신 결과가 화면에 표시된다.
```

### UC-03. Rate Limit 초과

```
1. 사용자가 하루 10회 분석 한도를 초과한다.
2. 서버에서 429 응답과 Retry-After 헤더를 반환한다.
3. 프론트엔드에 Rate Limit 모달이 표시된다.
4. 사용자는 '검색 히스토리 보기' 버튼을 통해 과거 결과를 확인할 수 있다.
```
