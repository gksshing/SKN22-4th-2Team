### ☁️ 클라우드 배포 아키텍처 상태 요약

(인프라 병목 구간 집중 추적)
현재 프론트엔드와 백엔드 레벨에서의 CORS 및 500 에러 처리 코드는 조치 완료되었습니다.
따라서 향후 브라우저에서 '네트워크 연결 오류'가 발생한다면 이는 100% 인프라 단의 병목 현상입니다. 문제 해결에만 중점을 두고, 즉시 다음과 같은 두 가지 핵심 모니터링을 관제하겠습니다.

1. **ALB 5xx 에러 트래픽 집중 감시:** 로드밸런서(ALB)의 응답 지연 타임아웃(504 Gateway Timeout) 방출 및 타겟 그룹 연결 실패(502 Bad Gateway) 메트릭을 `CloudWatch`에서 실시간으로 관찰합니다.
2. **ECS 컨테이너 OOM/CPU 스로틀링 즉각 점검:** 대규모 RAG 분석 요청 시 백엔드 컨테이너 리소스 초과로 인한 앱 정지(Exit Code 137 등) 현상이 없는지 `ECS Container Insights`를 통해 CPU 및 메모리 사용량의 스파이크 점검을 최우선으로 리포트합니다.
