### 2-1. [로딩/상태 UI] 로그인·회원가입 API 통신 대기 화면

#### 🎯 대상 시나리오
- 로그인 버튼 클릭 → `POST /api/v1/auth/login` 응답 대기
- 회원가입 버튼 클릭 → `POST /api/v1/auth/signup` 응답 대기

#### 📐 UI 설계

| 요소 | 설계 내용 |
|---|---|
| **버튼 상태** | 제출 중 버튼 비활성화(`disabled`) + 스피너 아이콘 표시 |
| **버튼 텍스트** | 로그인: `"로그인 중..."` / 회원가입: `"가입 중..."` |
| **전체 form 입력 비활성화** | `disabled` 속성으로 입력 수정 방지 (UX 혼선 차단) |
| **타임아웃 처리** | 15초 초과 시 "응답이 지연되고 있어요. 네트워크를 확인해주세요." 안내 |
| **스피너 스타일** | `animate-spin` 이모지 또는 SVG (⏳ → Tailwind CSS 기반) |

#### ✅ 현재 구현 상태 (Issue #46 완료 분)
- `LoginForm.tsx` / `SignupForm.tsx` — `isSubmitting` 상태로 버튼 스피너 **이미 구현됨**
- `isDisabled = isSubmitting || isLoading` 으로 전체 폼 잠금 **이미 구현됨**

> **추가 필요**: 15초 타임아웃 피드백 UI (기존 `TimeoutToast` 컴포넌트 패턴 재사용 권장)

### 2-2. [예외/에러 UI] 비정상 인증 시도 및 세션 만료 안내 모달

#### 🎯 에러 시나리오별 UI 설계

| 시나리오 | 에러 표시 방식 | 메시지 내용 |
|---|---|---|
| **로그인 실패** (이메일/비밀번호 불일치) | 인라인 에러 (form 내부) | "아이디 또는 비밀번호를 확인해주세요." |
| **회원가입 실패** (이메일 중복) | 인라인 에러 (email 필드 하단) | "이미 사용 중인 이메일입니다." |
| **네트워크 오류** (500, CORS, timeout) | 인라인 에러 (form submit 영역) | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |
| **세션 만료 (자동 로그아웃)** | **토스트 모달** (화면 상단/하단 고정) | "세션이 만료되었습니다. 다시 로그인해주세요." |
| **권한 없는 페이지 접근** | 페이지 레벨 에러 UI | "로그인이 필요한 페이지입니다." + 로그인 버튼 |

#### 📐 세션 만료 토스트 모달 설계

```
┌─────────────────────────────────────────┐
│ 🔒 세션이 만료되었습니다.               │
│    다시 로그인해주세요.    [로그인하기]  │
└─────────────────────────────────────────┘
```

- **트리거**: `auth:session-expired` CustomEvent (apiClient.ts에 이미 구현)
- **표시 위치**: 화면 하단 고정 (`fixed bottom-4`) 또는 상단 배너
- **자동 소멸**: 5초 후 자동 닫힘
- **액션 버튼**: "로그인하기" 클릭 → `/login` 이동
- **구현 방식**: 기존 `TimeoutToast` 컴포넌트를 확장하여 `SessionExpiredToast` 신규 생성 권장

#### 📐 계정 열거 공격 방어 원칙 유지
- 로그인 실패 시 이메일 존재 여부를 노출하는 에러 메시지 **절대 금지**
- 항상 단일 통합 메시지로 처리 (Issue #46 구현 내용 유지)

### 2-3. [사용자 플로우] 회원가입 폼 가독성 및 UX 개선

#### 📐 입력 필드 개선 사항

| 개선 항목 | 현재 | 개선안 |
|---|---|---|
| **비밀번호 복잡도 안내** | placeholder 텍스트만 표시 | 입력 시 실시간 복잡도 바 (4단계: 매우 약함/약함/보통/강함) |
| **비밀번호 표시/숨기기** | 숨김만 지원 | 👁️ 토글 버튼으로 비밀번호 확인 가능 |
| **비밀번호 일치 가시성** | onBlur 시 에러만 표시 | 일치 시 ✅ 녹색 아이콘, 불일치 시 ❌ 빨간 아이콘 실시간 표시 |
| **약관 동의 체크박스** | 없음 | 회원가입 완료 전 필수 약관 동의 체크박스 추가 |
| **입력 필드 순서** | 이메일 → 닉네임 → 비밀번호 → 확인 | 이메일 → 비밀번호 → 비밀번호 확인 → 닉네임 순서 권장 (자연스러운 흐름) |

#### 📐 회원가입 성공 후 플로우

```
회원가입 성공
  ↓
"회원가입이 완료되었습니다! 🎉" 성공 화면 (1~2초)
  ↓
로그인 화면 자동 이동 (또는 자동 로그인 처리)
```

## 5. Backend 협업 요청 사항

| 항목 | 요청 내용 |
|---|---|
| 이메일 중복 응답 | `HTTP 409 + { "detail": "EMAIL_ALREADY_EXISTS" }` 형태로 통일 |
| 로그인 실패 | `HTTP 401 + { "detail": "INVALID_CREDENTIALS" }` 단일 응답 유지 |
| API 응답 지연 | 정상 응답은 5초 이내 처리 (Auth 엔드포인트 SLA) |
